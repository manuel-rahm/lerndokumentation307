Kommentare

Kommentare sind Abschnitte im Skript, welche nicht ausgeführt werden und sind sinvoll für Angaben zum Skript, damit man leichter verstehen kann, was da nun gemacht wurde. Auch können Sie mit Kommentaren später besser nachvollziehen, was für Überlegungen Sie da gemacht haben.

Kommentare sind auch nützlich, um Code auszukommentieren, damit dieser nicht ausgeführt wird. Dies ist hilfreich, wenn man Code testen will, ob ein Code-Abschnitt überhaupt gebraucht wird.

Mehrzeilige Kommentare

Mehrzeilige Kommentare werden durch die Zeichenfolge /* eingeleitet und durch */ beendet. Alles was zwischen diesen beiden Zeichenfolgen steht, wird nicht als Quellcode interpretiert. Sie dürfen sich über mehrere Zeilen erstrecken, Eine Verschachtelung mehrzeiliger Kommentare ineinander ist nicht erlaubt.

/* Programm zur Berechnung von Lebensdauer Version vom 26.03.2019 Autor: Rias Gremory */

Einzeilige Kommentare

Einzeilige Kommentare werden mit // eingeleitet und erstrecken sich bis zum Ende der Zeile. Sie sind gut für kurze Erklärungen geeignet oder um einfach eine Zeile auszukommentieren.

Ein grosser Vorteil einzeiliger Kommentare ist, dass sie auch innerhalb von mehrzeiligen Kommentaren vorkommen können.

Bezeichner

Bezeichner sind Namen, die Sie für Funktionen, eigenen Objekte oder Variablen, vergeben.

Folgende Regeln sind bei der Namensvergabe zu beachten.

  • Sie dürfen keine Leerzeichen enthalten.
  • Sie dürfen nur aus Buchstaben und Ziffern bestehen - das erste Zeichen muss ein Buchstaben sein; Gross- und Kleinschreibung wird unterschieden.
  • Sie dürfen keine deutschen Umlaute oder scharfes S enthalten.
  • Sie dürfen als einziges Sonderzeichen den Unterstrich "_" enthalten.
  • Sie dürfen nicht mit einen reservierten Wort identisch sein.

Es empfiehlt sich sprechende Namen zu vergeben, die Ihnen auch längere Zeit später, noch signalisiert, welche Bedeutung sie haben.

Variablen

Eine Variable ist ein logischer Speicherplatz mit einem Namen und einem Wert, der verändert werden kann. In Javascript gibt es drei Möglichkeiten Variablen zu deklarieren.

  • var: deklariert eine Variable unabhängig von ihrem Gültigkeitsbereich.
  • let: deklariert eine lokale Variable im Block scope.
  • const: Symbolische Konstante; deklariert eine unveränderliche Variable.

var

Durch das Schlüsselwort var wird eine Variable vereinbart. Sie wird damit im aktuellen Scope angelegt. Ohne das Schlüsselwort wird die Variable im gloablen Scope erstellt.

Sie kann entweder ohne Wert deklariert werden:

var alter;

Oder der Anfangswert kann zu Beginn angegeben werden:

var alter = 18;

Anstelle eines "Werts" kann auch ein beliebiger Ausdruck stehen:

var Bezeichner = Ausdruck;

Beachten Sie:

  • Bei der Vergabe von Variablennamen gelten die Regeln für selbstvergebene Namen.
  • Beim Arbeiten mit Variablen sucht der Interpreter die Variable zuerst im lokalen und anschliessen im übergeordneten bis hin zum gloablen Sichtbarkeitsbereich.

Funktionen

Eine Funktion bezeichnet im Quellcode ein Stück Javascript, das an einer Stelle gespeichert ist und von anderen Stellen aus beliebig oft aufgerufen werden kann. Funktionen verfügen über Parameter, darunter versteht man eine Liste von Variablennamen, die während der Ausführung wie lokale Variablen zur Verfügung stehen und deren Werte - Argumente genannt - vom Aufrufen bereitgestellt werden. Eine Funktion lässt sich so leicht für unterschiedliche Zwekce aufrufen. Zum Abschluss kann eine Funktion einen einzelnen Wert an den Aufrufer zurückgeben.

Funktionsobjekte können auch Eigenschaft eines anderen Objektes sein, dann spricht man von Methoden.

Objektunabhängige Funktionen

Sie sind im Gegensatz zu selbstdefinierten Funktionen in Javascript bereits vordefiniert. Das heisst, Sie können diese Funktionen jederzeit aurufen, ohne sie vorher selbst definieren zu müssen, da sie bereits in Javascript intergriert sind.
