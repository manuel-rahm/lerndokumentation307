Document Object Model

Anhand des DOM können Browser ein Modell einer HTML-Seite erstellen, sodass Skripts auf die Inhalte dieser Webseite zugreifen und sie ändern können.

Das DOM stellt einen eignenen Satz von Regeln dar. Es wird von allen grossen Browserherstellern implementiert und ist für zwei Hauptaufgaben da:

Das Modell einer HTML-Seite erstellen

Wenn ein Browser eine Webseite lädt, erstellt er im Arbeitsspeicher ein Modell dieser Seite.

Das DOM gibt an, wie der Browser dieses Modell strukturieren soll, nämlich als DOM-Baum.

Das DOM wird als Objektmodell bezeichnet, da das Moddel aus Objekten besteht.

Jedes Objekt steht für einen Teil der Seite, die im Browserfenster geladen ist.

Elemente der HTML-Seite abrufen und ändern

Das DOM definiert auch Methoden und Eigenschaften, um auf die einzelnen Objekte in diesem Modell zuzugreifen und sie zu ändern. Dadurch können Sie ändern, was die Benutzer im Browser sehen.

Das DOM wird auch als Anwendungs-Programmerschnittstelle (API) bezeichnet. Über Benutzerschnittstellen können Menschen mit Programmen interagieren; über APIs können Programme miteinander kommunizieren. Das DOM bestimmt, was Ihr Skript den Browser über die aktuelle Seite fragen und wie es ihn anweisen kann, die Inhalte zu ändern.
